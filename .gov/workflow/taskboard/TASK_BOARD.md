## Usage Rules

1. Every meaningful change maps to a WP row.
2. Every WP row must have a linked test-suite file.
3. Every WP row must maintain linked `SX-WP-*` spec extraction and `check-WP-*` check-script references inside the WP file.
4. Update this board in the same PR as any WP status change.
5. `E2E-VERIFIED` is the only done status.
6. `SUPERSEDED` is allowed only for replaced packets and must cite the successor packet plus retained proof.
7. `E2E-VERIFIED` requires linked runtime evidence plus user sign-off.
8. Keep `Sub-Spec`, `Test Suite`, and `ROADMAP.md` links aligned for each iteration row.
9. Workflow Version `4.0+` packets must keep packet class, reality boundary, fallback register, and change ledger truthful in the linked WP artifacts.
